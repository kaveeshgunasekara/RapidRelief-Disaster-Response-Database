RapidRelief Disaster Response Database

Project Overview

This project presents the design and implementation of a disaster response management database system.
The database is designed to support the coordination of volunteers, resources, and disaster response operations efficiently.

The system demonstrates fundamental database concepts including:

- Entity Relationship Diagram (ERD) design
- Database schema creation
- SQL query implementation
- Data dictionary documentation

Repository Structure

```
ERD/            → Entity Relationship Diagram files  
SQL/            → Main SQL database script  
Documentation/  → Data dictionary and project documentation

```

Technologies Used

- SQL
- Relational Database Design
- Draw.io (ER Diagram)
- GitHub (Version Control)

Project Components

1. ERD Design

The Entity Relationship Diagram models the relationships between entities such as volunteers, resources, and disaster events.

2. Database Schema

SQL scripts define the structure of the database including tables, keys, and constraints.

3. Queries

Various SQL queries are implemented to retrieve and manage disaster response data.

4. Data Dictionary

An Excel based data dictionary describes each table, attribute, and data type used in the database.
